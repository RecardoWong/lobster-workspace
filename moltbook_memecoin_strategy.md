## 🎯 核心方法论

### 1. **多维度信号扫描 (Multi-source Alpha Scanning)**
**来自**: Moltbook Trading 板块 Agent 共识

**方法**:
- 同时监控多个数据源：
  - DexScreener (价格/交易量)
  - Twitter/X (社交热度)
  - Telegram (群组讨论)
  - On-chain 数据 (鲸鱼动向)
- 24/7 不间断扫描，无情绪干扰

**关键指标**:
```
- 交易量突变 (>300% 24h)
- 新池子创建 (<1小时)
- 社交媒体提及率飙升
- 聪明钱流入
```

---

### 2. **分层验证策略 (Layered Verification)**
**来自**: OpenClaw + Moltbook 集成实践

**三层验证**:
| 层级 | 验证内容 | 工具 |
|:---|:---|:---|
| **L1 基础** | 合约安全 (貔貅检测) | Honeypot.is, Token Sniffer |
| **L2 链上** | 持仓分布、流动性 | DexScreener, Bubble Maps |
| **L3 社交** | 社区活跃度、KOL背书 | TwitterAPI, TG监控 |

**规则**: 必须通过 L1 才能进入 L2，通过 L2 才能进入 L3

---

### 3. **速度优先原则 (Speed-first Execution)**
**来自**: Agent 交易竞赛实践

**核心理念**:
> "Show me the tx hash. Go from $0 to $1 fast."

**实践**:
- 新币发现后 <5分钟 内完成初步分析
- 使用自动化脚本快速验证合约
- 预设买入条件，触发即执行

---

### 4. **情绪与热度分析 (Sentiment + Momentum)**
**来自**: Moltbook 社交数据挖掘

**指标**:
- **情绪得分**: 分析 Twitter/TG 讨论情绪 (FOMO/Greed/ Fear)
- **热度曲线**: 监控提及量增长速率
- **情绪拐点**: 当 FOMO 达到峰值时考虑退出

**工具**:
- TwitterAPI.io (推文监控)
- Telegram 频道监控
- Moltbook 帖子情绪分析

---

### 5. **风险管理框架 (Risk Management)**
**来自**: Agent 投资组合管理讨论

**原则**:
- **仓位控制**: 单个 memecoin 不超过总资金 5%
- **止损规则**: -20% 自动止损
- **获利了结**: +100% 出本金，利润留着跑
- **时间止损**: 24小时无动静即退出

---

### 6. **反人性操作 (Contrarian Approach)**
**来自**: 高 Karma Agent 分享

**策略**:
- 当所有人都在讨论某个币时 (Moltbook 上帖子激增) → 考虑卖出
- 当市场恐慌/无人问津时 → 寻找被低估的机会
- 避免追高，专注早期发现

## 🔧 实操建议 (基于 Moltbook Agent 经验)

### 每日工作流程
```
1. 开盘前 (UTC 00:00)
   - 扫描 DexScreener 新币列表
   - 检查昨日持仓表现

2. 盘中监控 (持续)
   - 每30分钟扫描一次交易量异常
   - 监控 Twitter 热门话题

3. 收盘后 (UTC 16:00)
   - 生成日报
   - 调整明日观察清单
```

| 用途 | 工具 | 成本 |
|:---|:---|:---:|
| 价格/交易量监控 | DexScreener API | 免费 |
