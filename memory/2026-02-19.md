## Dashboard 修复（关键经验教训）

### 问题现象
- Twitter 动态区域一直显示"加载中..."
- 财经新闻和股价数据也加载失败
- 用户多次反馈数据不显示

### 根本原因
**浏览器缓存 + 动态加载策略问题**
1. 最初使用 JavaScript `fetch()` 动态从 JSON 文件加载 Twitter 数据
2. 浏览器缓存了旧的 JS 代码或 JSON 数据
3. 时区格式化代码 `toLocaleString('zh-CN')` 在某些浏览器中报错
4. 导致页面卡在"加载中..."状态

### 解决方案
**改为静态嵌入策略：**
1. Twitter 推文数据直接写在 HTML 中（静态嵌入）
2. 不再依赖 JavaScript 动态加载
3. 更新时间直接显示在 HTML 中
4. 财经新闻仍使用 JSON + JavaScript 加载（数据量大，适合动态）

### 实施步骤
1. 创建 `twitter_auto_deploy.py` 脚本
2. 每小时自动抓取 Twitter 最新推文
3. 使用 MyMemory API 自动翻译
4. 生成包含最新数据的 HTML 文件
5. 自动部署到服务器

### 关键教训（必须记住）
- **不要纯依赖 JavaScript 动态加载关键数据**
- **静态嵌入 + 定时更新** 比纯动态加载更可靠
- **浏览器缓存** 是前端问题的常见原因
- **强制刷新 (Ctrl+F5)** 可解决大部分缓存问题
- **避免使用浏览器特定的 API**（如 `toLocaleString` 时区格式化）

## Dashboard v4.0 迭代总结 (17:23 完成)

### 本次迭代解决的问题
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Twitter显示"加载中" | 浏览器缓存JS | 静态HTML嵌入 |
| 股价数据是假的 | 用了静态默认值 | 接入腾讯财经/币安API |

| 更新时间不显示 | 硬编码时间标签 | 动态更新时间戳 |

### 关键教训
1. **不要用假数据糊弄用户** - 被发现后会失去信任，应诚实说明限制
2. **静态嵌入 > 纯动态加载** - 关键数据直接写HTML，避免浏览器缓存问题
3. **腾讯财经API很稳定** - us.IXIC(纳斯达克)、hkHSTECH(恒生科技)都能获取
4. **正则替换要小心** - 容易破坏HTML结构，导致页面错乱
5. **用户反馈是改进来源** - 序号、红色时间、数据中心新闻都来自用户要求
