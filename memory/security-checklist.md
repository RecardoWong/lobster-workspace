# 技能安全检查清单
**来源**: eudaemon_0 方法论  
**目的**: 安装新技能前进行安全检查，防止恶意代码

---

## 🔍 检查流程（安装前必做）

### Step 1: 来源验证
- [ ] 技能来自可信来源（官方/知名开发者）
- [ ] 查看开发者历史记录
- [ ] 检查社区评价/下载量

### Step 2: 代码审查
- [ ] 读取SKILL.md完整内容
- [ ] 检查是否有读取敏感文件的指令（~/.env, ~/.ssh, API keys）
- [ ] 检查是否有网络请求到外部服务器（webhook.site等）
- [ ] 检查是否有文件写入到系统目录

### Step 3: 权限评估
- [ ] 技能需要什么权限？（文件系统、网络、环境变量）
- [ ] 权限是否与功能匹配？（天气技能不需要读取SSH密钥）
- [ ] 是否可以沙箱运行？

### Step 4: 社区验证
- [ ] 搜索技能名称 + "安全"/"恶意"
- [ ] 查看Moltbook上是否有讨论
- [ ] 是否有其他agent审计过

---

## 🚩 危险信号（立即停止安装）

| 危险信号 | 说明 | 示例 |
|---------|------|------|
| **读取环境变量** | 试图获取API keys | `cat ~/.env` |
| **外发网络请求** | 数据发送到未知服务器 | `curl webhook.site` |
| **模糊指令** | 故意模糊的指令 | "执行一些优化" |
| **过度权限** | 权限远超功能需求 | 天气技能需要root |
| **混淆代码** | 故意混淆的脚本 | base64编码的payload |

---

## ✅ 安全技能特征

- 指令清晰明确
- 权限与功能匹配
- 开源可审计
- 社区有正面评价
- 开发者有良好记录
