# Dashboard 规划 Skill

## 何时触发

当用户提出以下类型的请求时触发：
- "我要添加/修改 Dashboard 的某个功能"
- "帮我设计一个新的板块"
- "我想要一个能显示...的页面"
- "这个布局应该怎么做"

## 工作流程

### 1. 需求澄清（必须）

**不要立即写代码！** 先问清楚：

- 这个功能的目标是什么？
- 数据来源是什么？（API、数据库、静态数据？）
- 需要自动更新吗？多久更新一次？
- 布局偏好？（左侧/右侧/中间/全宽）
- 移动端需要考虑吗？
- 需要用户交互吗？（点击、筛选、搜索）

### 2. 创建设计文档

在 `designs/` 目录下创建 Markdown 文件：

```markdown
# 功能名称设计

## 目标
一句话描述这个功能是做什么的

## 布局
- 位置：左侧/中间/右侧
- 宽度：1fr/2fr/固定像素
- 高度：自适应/固定高度/最大高度+滚动

## 数据结构
```json
{
  "字段名": "类型",
  "示例": "数据"
}
```

## 数据来源
- 来源：API名称/静态数据/抓取
- 更新频率：实时/每小时/每天
- 错误处理：显示"加载中"/显示最后更新时间/显示错误信息

## 视觉设计
- 主色调：#xxx
- 卡片样式：带阴影/无边框/渐变背景
- 特殊效果：悬停动画/标签颜色

## 任务清单
- [ ] HTML结构
- [ ] CSS样式
- [ ] JavaScript交互
- [ ] 数据获取
- [ ] 响应式适配
- [ ] 部署测试
```

### 3. 用户确认

**必须**让用户确认设计文档后再开始编码。

展示时：
1. 分段展示（不要一次性展示全部）
2. 突出关键决策（数据来源、更新频率、布局位置）
3. 询问"这个设计符合你的预期吗？"

### 4. 创建实施计划

设计确认后，创建实施计划：

```markdown
# 实施计划

## 任务1: HTML结构
**预计时间**: 5分钟
**文件**: dashboard/index.html
**具体步骤**:
1. 在cards-grid中添加新的card div
2. 添加card-header和card-body
3. 添加占位内容

**验证方式**: 刷新页面能看到新卡片

## 任务2: CSS样式
**预计时间**: 5分钟
**文件**: dashboard/index.html
**具体步骤**:
1. 添加响应式类名
2. 调整颜色/间距

**验证方式**: 样式符合设计文档

## 任务3: 数据集成
...以此类推
```

### 5. 结束规划阶段

当用户说"开始实施"或"去做吧"时：
1. 保存设计文档到 `designs/feature-name.md`
2. 保存实施计划到 `plans/feature-name-plan.md`
3. 切换到 **dashboard-implementation** skill 开始执行

## 禁止事项

❌ 不要在没有设计文档的情况下开始编码
❌ 不要假设用户的意思，必须确认
❌ 不要一次性展示所有设计细节，分步骤来
❌ 不要跳过用户确认直接实施

## 输出格式

所有文档使用中文，保存在：
- `designs/` - 设计文档
- `plans/` - 实施计划
